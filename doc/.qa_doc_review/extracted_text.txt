基于 RT-Thread 的智能髋膝关节假肢系统
参赛赛道：RT-Thread 自主命题赛道
摘要
我国下肢截肢患者超300万，且每年以10万以上的速度持续增长。其中因创伤、糖尿病足、血管病变导致的下肢关节功能受限占比超过65%。然而，当前市场上智能假肢产品面临严峻困境：进口产品技术成熟但价格高达10—30万元，医保覆盖有限，普通患者难以承受；传统被动假肢无动力、无感知，仅能实现基础支撑，无法适配上下楼梯、斜坡、凹凸路面等复杂场景，患者生活质量严重受限。
本项目面向髋膝关节假肢的实时感知、状态监测与电机协同控制需求，构建了一套基于 RT-Thread 的分布式感知与电机主控系统。系统以 STM32F407 为核心控制器，在 RT-Thread 线程模型下并行运行 ADS1194 生物电采集任务、BMI088 姿态检测任务和 motor_ctrl 电机主控任务：ADS1194 完成四通道 ECG/EMG 原始信号采集、寄存器校验、毫伏换算和数字滤波；BMI088 完成加速度、角速度、姿态角、温度和跌倒状态计算；电机主控程序根据肌电强度、惯性姿态和步态相位生成髋膝双关节目标角，并通过 CAN/RS485 通道下发到电机执行端。STM32 同时将四通道波形、三轴陀螺仪、姿态角、温度、跌倒标志和控制状态通过 USART2 实时发送至 ESP32-S3，ESP32 负责 Wi-Fi/WebSocket 转发、LCD 调试显示和浏览器端波形展示，本地 Python 上位机提供任务记录、CSV 保存、历史回放和导出。项目突出 RT-Thread 在多线程任务调度、外设协同、实时数据链路和电机控制任务组织中的作用，符合 RT-Thread 自主命题赛道对自选场景、开源嵌入式系统应用和工程完整性的要求。
赛道适配说明：本作品以“髋膝关节假肢实时感知、状态监测与电机协同控制”为自主命题场景，核心代码按 RT-Thread + STM32F407 平台组织，体现 RT-Thread 在实时采集、多任务调度、电机主控和嵌入式系统集成中的应用价值。
第一部分  作品概述
1.1 功能与特性
本系统是一款面向髋膝关节假肢的实时感知、监测与电机主控平台，围绕四项核心功能构建：（1）多通道生物电采集与滤波：STM32F407 通过 SPI3 驱动 ADS1194，完成四通道 ECG/EMG 原始数据读取、状态字校验、毫伏换算和滤波处理，为运动意图分析与生理状态监测提供基础数据；（2）大腿姿态与安全状态检测：通过 SPI1 驱动 BMI088，实时计算 roll、pitch、yaw、温度和跌倒标志，并按照大腿安装方向定义屈伸角；（3）RT-Thread 电机主控程序：以 motor_ctrl 线程形式完成上电校准、IDLE/MOVE/HOLD/LOST 状态管理、步态相位推进、双关节目标角生成和限速保护，通过 CAN/RS485 控制髋、膝关节 GO-M8010-6 电机；（4）无线显示与数据留存：STM32 通过 USART2 向 ESP32-S3 输出统一 CSV 行，ESP32 解析后经 WebSocket 推送到浏览器界面，同时本地上位机可完成采集任务管理、CSV 保存、历史回放和导出。
1.2 应用领域
（1）下肢截肢患者康复：本产品的核心应用领域为下肢截肢患者的日常行走康复与生活辅助。我国该群体规模约200万人（占下肢截肢患者300万的三分之二以上），且以每年7万人以上的速度增长。糖尿病足导致的非创伤性截肢、交通事故与工伤事故导致的创伤性截肢是两大主要成因。（2）康复医疗机构的评估与训练：康复医院、假肢装配中心可利用本系统的上位机显示功能，在适配和训练过程中实时观察患者的肌电信号特征和心电状态，辅助技师进行精准的假肢参数调校和康复方案制定。（3）可扩展至小腿截肢领域：本产品的髋膝关节动力模块采用模块化设计，可集成于小腿假肢系统中，服务于更大范围的截肢患者群体。（4）老龄化社会的康复需求：我国老龄化进程加速，因血管疾病导致截肢的老年患者数量持续上升。智能假肢的防摔倒功能和生理信号监测能力，对平衡能力较差的老年患者尤为重要。
1.3 主要技术特点
（1）RT-Thread 多线程嵌入式软件架构：STM32F407 在 RT-Thread 上分别创建 ADS1194 采集线程、BMI088 姿态线程和 motor_ctrl 电机主控线程，使生物电采集、姿态更新、步态控制和上位机数据输出互不阻塞。 （2）ADS1194 四通道生物电采集链路：代码实现芯片复位、ID 校验、寄存器配置、RDATA 帧读取、状态字判断、16 位通道解码、毫伏换算和滤波处理。 （3）BMI088 姿态与跌倒检测：代码完成加速度计和陀螺仪初始化，配置加速度 6g、陀螺仪 500dps，并计算 roll/pitch/yaw、屈伸角、温度和跌倒状态。 （4）双关节电机主控程序：代码按 RT-Thread 任务化方式组织 CAN2 肌电/IMU 输入和 CAN/RS485 髋膝电机控制，完成静止标定、肌电触发、步态相位冻结/推进、目标角曲线插值和执行端异常保持。 （5）STM32-ESP32-上位机数据链路：STM32 以 1Mbps USART2 输出四通道波形、三轴角速度、跌倒标志、姿态角、温度和控制状态，ESP32 解析后通过 WebSocket 批量推送，浏览器端实现实时波形、状态显示、CSV 保存和历史回放。
1.4 主要性能指标
硬件尺寸：分布式感知节点尺寸≤20mm×30mm，适配假肢接受腔安装空间；
信号处理：sEMG信号处理信噪比≥25dB，误触发率≤2%；
控制精度：髋膝关节运动姿态追踪误差≤0.5°，电机角度控制误差≤0.5°；
实时性：系统决策周期≤300ms（人体神经肌肉控制感知阈值）；
通信性能：STM32 与 ESP32 串口链路配置为 1Mbps，ESP32 到浏览器端采用 WebSocket 批量推送；
功耗：系统待机功耗≤50mA，工作功耗≤200mA，满足锂电池供电的穿戴需求。
1.5 主要创新点（1）面向 RT-Thread 自主命题赛道的髋膝关节假肢应用：将 RT-Thread 用于多传感器实时采集、线程调度、电机主控和外设协同，形成可演示的康复工程场景。 （2）生物电、惯性姿态与电机控制一体化数据链路：在 STM32 端将 ADS1194 四通道 ECG/EMG 数据、BMI088 姿态/温度/跌倒标志和 motor_ctrl 控制状态合并为统一数据流，降低 ESP32 和上位机解析复杂度。 （3）RT-Thread 化双关节电机主控：将 CAN/RS485 髋膝电机和 CAN2 肌电/IMU 输入组织为同一控制任务，实现 IDLE/MOVE/HOLD/LOST 状态切换和步态相位协调。 （4）轻量化无线监测链路：ESP32-S3 负责 Wi-Fi/WebSocket 转发和 LCD 调试显示，浏览器端无需专用客户端即可查看实时波形并保存 CSV。 （5）可复用的嵌入式调试路径：USART1/USART2/USART3 与 CAN 计数状态保留调试输出，便于硬件联调和故障定位。
（1）机电与心电一体化采集方案：基于三导联配置实现EMG与ECG信号同步采集，将运动意图识别与生理状态监测融合于同一硬件平台，提升了系统的集成度和功能完整性。（2）分布式感知与双路隔离CAN通信架构：设计物理隔离的双通道通信体系，将实时控制数据与状态监测数据分流传输，结合CAN总线与卡尔曼滤波实现多传感器高可靠低延迟融合。（3）上位机实时监测系统：开发专用PC端软件，提供肌电信号、心电图及步态参数的实时可视化显示，服务于临床调试与康复评估。（4）自研滑模控制算法的假肢应用：将SMC算法应用于髋膝关节假肢控制，配合在线学习与多地形自适应策略，实现智能行走控制。
1.6 设计流程
系统设计遵循“需求分析→RT-Thread 嵌入式软件开发→传感器驱动调试→电机主控程序开发→STM32/ESP32 通信→上位机可视化→系统联调验证”的流程。首先明确髋膝关节假肢对生物电、姿态、安全状态和双关节执行控制的需求；随后在 STM32F407 上完成 CubeMX 外设配置，并在 RT-Thread 下实现 ADS1194、BMI088 和 motor_ctrl 三个核心任务；接着统一 STM32 输出数据格式，通过 USART2 与 ESP32-S3 对接；再开发 WebSocket 浏览器界面和 Python 本地服务，实现实时显示、CSV 保存和历史回放；最后通过串口日志、网页波形、CAN/RS485 执行状态和本地记录验证数据链路。
第二部分  系统组成及功能说明
2.1 系统整体架构本系统的整体架构由感知层、RT-Thread 控制层、电机执行层、通信转发层和上位机显示层构成。感知层包括 ADS1194 生物电采集前端和 BMI088 惯性测量单元；RT-Thread 控制层运行在 STM32F407 上，负责传感器初始化、周期采集、滤波、姿态计算、跌倒判断、步态相位管理和髋膝目标角生成；电机执行层包括髋关节电机和膝关节电机，接收 motor_ctrl 线程下发的目标位置、速度和保护参数；通信转发层由 USART2 与 ESP32-S3 组成，ESP32 解析 STM32 数据后通过 Wi-Fi/WebSocket 推送；上位机显示层由浏览器界面和 Python 本地服务组成，提供实时波形、控制状态、采集任务、CSV 保存、历史回放和导出。
2.2 分层功能说明感知层：ADS1194 采集四通道 ECG/EMG 原始数据，BMI088 采集大腿姿态与温度。RT-Thread 控制层：STM32F407 初始化 GPIO、USART、SPI、CAN 等外设后启动 ads1194、bmi088 和 motor_ctrl 线程，分别完成生物电采集、姿态更新和髋膝电机主控。电机执行层：motor_ctrl 线程根据肌电阈值、姿态强度和步态相位计算髋、膝目标角，并通过 CAN/RS485 完成髋关节和膝关节电机控制。通信层：STM32 以统一 CSV 行经 USART2 输出到 ESP32-S3，ESP32 负责 WebSocket 服务和可选 LCD 调试显示。上位机层：浏览器显示 ECG、EMG、姿态角、温度、跌倒状态和控制状态，Python 服务保存采集任务和 CSV 数据。
系统整体框图应突出“ADS1194/BMI088 → STM32F407 + RT-Thread → motor_ctrl → CAN/RS485 → 髋/膝电机执行端”的控制主链路，同时保留“STM32F407 → USART2 → ESP32-S3 → WebSocket/浏览器/CSV”的监测支路。

2.3 硬件系统组成硬件系统由生物电采集模块、惯性姿态模块、STM32F407 主控模块、髋膝电机执行模块、ESP32-S3 无线显示模块和电源/接口模块组成。（1）生物电采集模块：以 ADS1194 为核心，通过 SPI3 与 STM32 通信，完成四通道 ECG/EMG 数据采集。（2）惯性姿态模块：以 BMI088 为核心，通过 SPI1 与 STM32 通信，输出加速度、角速度和温度数据。（3）主控模块：STM32F407 运行 RT-Thread，负责传感器驱动、线程调度、滤波处理、姿态与跌倒状态计算和 motor_ctrl 任务调度。（4）电机执行模块：髋关节与膝关节电机通过 CAN/RS485 接入，主控程序完成目标角、速度和保护状态下发。（5）无线显示模块：ESP32-S3 接收 STM32 串口数据，提供 Wi-Fi/WebSocket 转发和 LCD 调试显示。
2.4 软件系统组成软件系统由 STM32 RT-Thread 固件、RT-Thread 化 motor_ctrl 电机主控程序、ESP32-S3 转发固件和 PC 端 Web 上位机构成。STM32 端在 main.c 中初始化 GPIO、USART1、USART2、SPI1、SPI3、CAN 等外设后启动 bmi088_start()、eeg_raw_stream_init() 和 motor_ctrl 任务；BMI088 线程完成姿态角、温度和跌倒检测，ADS1194 线程完成四通道采样、滤波和数据帧输出，motor_ctrl 线程完成肌电触发、步态相位推进、髋膝目标角插值以及 CAN/RS485 电机控制。ESP32 端初始化 STM32 串口、Wi-Fi 和 WebSocket 服务，解析 STM32 的 CSV 数据行，并推送 ECG、EMG、陀螺仪、姿态角、温度、跌倒状态和控制状态。PC 端 web-monitor 采用轻量化 Python HTTP/API 服务，浏览器直接显示实时波形，同时将任务数据保存为 CSV，支持历史查询、回放、导出和删除。
软件模块说明：STM32 端包括 main.c 外设入口、eeg_raw_stream.c 生物电采集与滤波、bmi088.c 姿态/跌倒检测、filter.c 数字滤波、motor_ctrl 电机主控、can.c/getdata.c CAN 接收与电机反馈、GO-M8010-6.c RS485 电机协议封装；ESP32 端包括串口解析、WebSocket 批量推送、LCD/编码器调试界面；PC 端包括 server.py 任务 API、index.html/app.js/styles.css 实时显示与历史回放界面。

第四部分  总结
可扩展之处
本系统在 RT-Thread 线程模型、传感器接口、电机控制接口和通信链路上均预留了扩展空间。
其一，当前 ADS1194 四通道采集方案可继续扩展到更多导联或更复杂的运动意图识别算法，以提升髋膝关节控制输入的可靠性。
其二，STM32F407 的 RT-Thread 软件框架可继续优化 motor_ctrl 电机控制线程、CAN 通信线程和参数管理线程，将现有感知-控制链路扩展为更完整的闭环控制系统。
其三，ESP32/WebSocket 上位机链路可接入训练记录、远程康复评估或移动端显示，实现更完整的数据管理。
其四，现有姿态角、跌倒检测和多通道生物电数据可作为后续步态识别、滑模控制或自适应控制算法的输入，逐步从监测样机扩展到主动驱动样机。
心得体会
回顾整个研发周期，从最初的概念论证到最终的样机联调，每一步都充满挑战，也让我对“智能假肢”这一交叉学科领域有了远超越书本的深刻理解。
最核心的感悟在于：工程系统的复杂性不在于单个模块的极致性能，而在于模块之间如何可靠地协同。 我们的系统涉及微弱模拟信号采集、数字通信、实时控制算法、电机驱动和上位机可视化五个层面，任何一个环节的噪声、延迟或数据丢失，都会在闭环中迅速放大，最终表现为髋膝关节的抖动或响应滞后。比如，在早期联调时，肌电信号采集正常、CAN通信误码率正常、电机空载响应也正常，但一旦接入假肢机械结构进行带载行走测试，系统就会出现偶发性力矩波动。排查持续了近两周，最终发现根本原因并非算法参数，而是电源模块的瞬态响应不足——当电机急加速时，电池电压跌落导致ADC参考电压波动，进而使肌电采样产生偏置漂移。这个“小问题”让我深刻意识到，硬件设计中的电源完整性、信号地回路和隔离设计，其重要性丝毫不亚于控制算法本身。
在算法层面，滑模控制器的实际调试远比仿真复杂。仿真中理想的滑模面在真实系统上会遇到传感器噪声、机械间隙和非线性摩擦的综合干扰，抖振现象尤其棘手。我们花了大量时间在符号函数替换为饱和函数、边界层厚度的自适应调整上，最终才在鲁棒性和舒适性之间取得平衡。这让我明白，控制理论的实际落地，需要工程师对物理系统有深刻的直觉，而不仅仅是数学推导。
团队协作方面，由于项目横跨电子、机械、控制和软件，不同专业背景的成员在沟通时经常出现“术语壁垒”。曾经机械组的成员描述“髋膝关节自由度不足”，控制组理解成了“控制量维度过低”，双方争论了两天才发现指的是完全不同的两个问题。后来我们建立了统一的接口文档和信号流图，将所有关键参数的定义、单位和取值范围书面化，才有效降低了沟通成本。这段经历让我深刻体会到，跨学科团队最宝贵的不是各自的高深技能，而是建立共同语言和系统级思维的能力。
此外，上位机软件的开发也让我意识到“可视化”在工程调试中的巨大价值。在早期阶段，我们只能通过串口打印数值来判断系统状态，效率极低；一旦开发出波形显示界面，很多问题（如肌电基线漂移、心电工频干扰、步态相位切换滞后）都能一目了然地被观察到。工具链的完善，往往能反向推动问题的解决速度。
最后，每次看到团队成员在深夜的实验室里，一次又一次地让假肢在跑步机上从磕绊走向平稳，那种“从0到1”的成就感是难以替代的。智能假肢不仅仅是机电系统的集成，它承载的是截肢患者重新获得行走尊严的期望。虽然当前系统距离商业化产品还有可靠性和轻量化方面的距离，但我们已经走出了扎实的第一步。未来，我会继续深耕康复工程领域，让更多患者受益于中国智造的技术红利。
