# BioScope 项目需求与数据协议

> 状态：需求确认中  
> 版本：0.1  
> 本文件是后续开发和其他 agent 的统一需求来源。实现前应先阅读本文件；若用户给出新要求，以用户最新要求为准并同步更新本文件。

## 1. 已确认需求

| 项目 | 要求 |
| --- | --- |
| 运行位置 | 电脑本地网站 |
| 数据连接 | WebSocket |
| 心电通道 | 1 路 ECG |
| 肌电通道 | 3 路 EMG |
| 采样率 | 四个通道均为 500 Hz |
| 数据保存 | 仅保存为 CSV，不使用数据库 |
| 历史回放 | 按采集任务查询和回放 |
| 数据导出 | CSV 导出 |
| 异常报警 | 心率过高、心率过低、心电异常、肌电超限、信号断开 |
| 报警阈值 | 用户可在网页中设置 |
| 用户系统 | 不需要登录和患者管理 |
| 移动端 | 不需要专门适配手机和平板 |
| 任务控制 | 用户手动开始、停止 |
| CSV 写入 | 本地后端实时写入指定文件夹 |
| 心电异常 | 演示规则检测心率不齐、漏搏、信号幅度异常 |
| 报警表现 | 红色提示、波形时间点标记；不要求声音 |
| 默认心率阈值 | 低于 50 BPM、高于 120 BPM |
| 暂停行为 | 暂停波形时不继续后台记录 |
| 任务信息 | 需要任务名称和备注 |
| 默认保存目录 | 项目内 `data` 文件夹 |
| 历史保留 | 永久保留，由用户手动删除 |
| EMG 默认阈值 | EMG1–3 均为 0.5 mV RMS |
| ECG 幅度阈值 | 超出 ±2 mV 标记异常 |
| 断线报警 | 连续 2 秒未收到数据 |
| 单任务限制 | 不限制采集时长 |

## 2. 暂定系统结构

```text
STM32/采集前端
      |
      | 4 通道、500 Hz
      v
    ESP32
      |
      | WebSocket
      v
电脑本地 BioScope 网站
      |
      +-- 实时波形与指标
      +-- 报警判断
      +-- 采集任务管理
      +-- 本地后端实时写 CSV
      +-- 历史任务与波形回放
```

- 默认 WebSocket 地址：`ws://192.168.4.1/ws`
- 网站运行在电脑本地，不要求由 ESP32 托管网页文件。
- 本地后端负责接收或转发 WebSocket 数据、实时写入 CSV、列出历史任务及读取回放数据。
- 当前协议为前后端联调的暂定协议，ESP32 固件实现时应遵循此定义。

## 3. WebSocket 数据协议

### 3.1 推荐格式：批量 JSON

ESP32 每次发送一批连续样本，推荐每帧包含 10 个样本，即 500 Hz 时每 20 ms 发送一次。

```json
{
  "type": "samples",
  "version": 1,
  "deviceId": "bioscope-esp32-01",
  "sampleRate": 500,
  "seqStart": 12500,
  "timestampUs": 1710000000000000,
  "samples": [
    [0.421, 0.115, 0.083, 0.132, 1.2, -0.4, 0.1, 0, 2.1, 84.3, 0.5, 31.4],
    [0.438, 0.121, 0.079, 0.128, 1.1, -0.5, 0.2, 0, 2.2, 84.2, 0.6, 31.4],
    [0.405, 0.118, 0.086, 0.135, 40.0, 12.0, -8.0, 1, 58.0, 20.0, 6.8, 31.5]
  ]
}
```

每个样本数组的顺序固定为：

```text
[ecg, emg1, emg2, emg3, gyro_x, gyro_y, gyro_z, fall, roll, pitch, yaw, temperature]
```

字段定义：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | string | 固定为 `samples` |
| `version` | integer | 协议版本，当前为 `1` |
| `deviceId` | string | ESP32 设备标识 |
| `sampleRate` | integer | 每通道采样率，固定为 `500` |
| `seqStart` | unsigned integer | 本帧首个样本的连续序号，用于检测丢包 |
| `timestampUs` | unsigned integer | 本帧首个样本时间戳，单位微秒 |
| `samples` | array | 一批四通道样本和运动数据 |

数值单位暂定：

- ECG：mV
- EMG1、EMG2、EMG3：mV
- 时间戳：µs

### 3.2 设备状态消息

ESP32 可定期发送状态消息：

```json
{
  "type": "status",
  "version": 1,
  "deviceId": "bioscope-esp32-01",
  "uptimeMs": 360000,
  "sampleRate": 500,
  "bufferUsage": 12,
  "rssi": -48
}
```

### 3.3 连接与断线规则

- WebSocket 建立成功不等于已有有效数据。
- 网站收到第一帧合法 `samples` 消息后，才进入“正在采集”状态。
- 超过可配置时间未收到合法样本时触发“信号断开”报警。
- 网站利用 `seqStart` 检测缺失样本并统计丢包率。
- 不允许网站在未连接时自动产生真实数据显示；模拟数据必须由用户明确启动。

## 4. CSV 文件定义

每个采集任务保存为一个 CSV 文件。建议文件名：

```text
任务名称_YYYYMMDD_HHMMSS.csv
```

CSV 使用 UTF-8 编码，首行为列名：

```csv
sample_index,timestamp_us,ecg_mv,emg1_mv,emg2_mv,emg3_mv,gyro_x_dps,gyro_y_dps,gyro_z_dps,fall,roll_deg,pitch_deg,yaw_deg,temperature_c,alarm_flags
0,1710000000000000,0.421,0.115,0.083,0.132,1.2,-0.4,0.1,0,2.1,84.3,0.5,31.4,0
1,1710000000002000,0.438,0.121,0.079,0.128,40.0,12.0,-8.0,1,58.0,20.0,6.8,31.5,128
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `sample_index` | 本任务内从 0 开始的样本序号 |
| `timestamp_us` | 样本时间戳，单位微秒 |
| `ecg_mv` | ECG，单位 mV |
| `emg1_mv` | EMG1，单位 mV |
| `emg2_mv` | EMG2，单位 mV |
| `emg3_mv` | EMG3，单位 mV |
| `gyro_x_dps` | X 轴角速度，单位 dps |
| `gyro_y_dps` | Y 轴角速度，单位 dps |
| `gyro_z_dps` | Z 轴角速度，单位 dps |
| `fall` | 跌倒判断，0 为正常，1 为跌倒 |
| `roll_deg` | 大腿横滚角，单位 deg |
| `pitch_deg` | 大腿俯仰角，单位 deg |
| `yaw_deg` | 大腿偏航角，单位 deg |
| `temperature_c` | BMI088 温度，单位 C |
| `alarm_flags` | 当前样本对应的报警位标记 |

报警位暂定：

| Bit | 数值 | 含义 |
| ---: | ---: | --- |
| 0 | 1 | 心率过低 |
| 1 | 2 | 心率过高 |
| 2 | 4 | 心电异常 |
| 3 | 8 | EMG1 超限 |
| 4 | 16 | EMG2 超限 |
| 5 | 32 | EMG3 超限 |
| 6 | 64 | 信号断开 |
| 7 | 128 | 跌倒警告 |

## 5. 采集任务

每次采集必须属于一个任务。任务至少包含：

- 任务名称
- 任务备注
- 创建时间
- 开始时间
- 结束时间
- 采集时长
- 样本总数
- CSV 文件
- 报警次数与类型汇总

历史页面按任务展示，并支持：

- 查看任务概要
- 打开四通道历史波形
- 按时间拖动和缩放回放
- 查看报警时间点
- 导出或下载原始 CSV
- 删除任务（删除前必须确认）

## 6. 报警设置

网页必须提供可编辑设置，至少包括：

- 心率下限，单位 BPM
- 心率上限，单位 BPM
- ECG 异常检测开关
- EMG1 RMS 上限，单位 mV
- EMG2 RMS 上限，单位 mV
- EMG3 RMS 上限，单位 mV
- 断线判定时间，单位 ms
- 报警声音开关

当前已确认的默认心率阈值：

- 心率过低：低于 50 BPM
- 心率过高：高于 120 BPM
- ECG 幅度异常：超出 ±2 mV
- EMG1 RMS 过高：高于 0.5 mV
- EMG2 RMS 过高：高于 0.5 mV
- EMG3 RMS 过高：高于 0.5 mV
- 信号断开：连续 2 秒未收到有效样本

报警界面需要：

- 显示醒目的红色提示
- 在实时波形和历史波形中标出报警时间点
- 不要求声音报警

心电异常检测暂定为工程演示规则，包括：

- 心率不齐
- 漏搏
- ECG 信号幅度异常

这些规则不作为医学诊断结论。

设置保存在浏览器本地，不需要账号或数据库。
